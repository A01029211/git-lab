Que es Git?

Git es un sistema de control de versiones que realiza un seguimiento de los cambios en los archivos. Git es espacificamente util cuando se trabaja en equipo ya que estas haciendo cambios en los mismos archivos al mismo tiempo

Que es GitHub?

GitHub es una plataforma basada en la nube donde puedes almacenar, compartir y tabajar junto con otros usuarios para escribir un codigo o hacer un proyecto

Que son los commits?

Los commits guardan el estado actual de tus archivos para poder volver a ellso en determinado punto si es que es necesario. Con los comits es facil ver que cambio se hizo para que si hay un error puedes checar en los commits para ver si alguno de ellos pudo haber causado el error.

Comandos en terminal para Git

Comandos de Git

Propositos
git clone Git clone hace que se copie el repositorio de alguien mas para ponerlo en tu propio repositorio o en donde el usuario guste
git branch -b "" con este comando se crea una nueva rama en donde el usuario puede trabajar
git pull El git pull agarra todos los cambios que se hicieron en un repositorio para asegurar que estaras trabajando con todos los archivos y datos correctos
git add cuando hayas terminado de hacer tus cambios se utiliza el comando git add acompañado del nombre del archivo para agregar el archivo editado a tu repositorio.
git commit -m "" Cuando terminas de hacer el git add haces el git commit -m acompañado del mensaje que quieres mandar para identificaar el cambio que hiciste al archivo o repositoro
git push -u origin Cuando termines tu commit tienes que hacer el git push para que se manden todos los cambios al repositorio y se guarden el la nube
git merge si estas trabajando en una rama diferente es necesario hacer e git merge para que se incorporen tus cambios a la rama principal
